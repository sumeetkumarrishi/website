Understanding Retrieval Augmented Generation (RAG)

Category: AI → RAG

Introduction

Large Language Models (LLMs) possess vast amounts of knowledge learned during training. However, they have limitations. Their knowledge may be outdated, they may not be aware of company-specific information, and retraining them whenever new information becomes available is expensive and time-consuming.

Retrieval Augmented Generation (RAG) solves this problem by combining information retrieval with language generation. Instead of relying solely on what the model learned during training, relevant information is retrieved from an external knowledge source and provided to the model when a question is asked.

What is RAG?

RAG stands for Retrieval Augmented Generation.

It is a technique that enhances an LLM's responses by supplying relevant information from an external knowledge base before generating an answer.

The idea is simple:

  1. A user asks a question.
  2. Relevant information is retrieved from a knowledge source.
  3. The retrieved information is added to the prompt.
  4. The LLM generates an answer using both the retrieved information and its existing knowledge.

This allows the model to answer questions using information it was never explicitly trained on.

Why RAG is Important

Modern LLMs are powerful, but they face several challenges:

  • Knowledge becomes outdated after training.
  • Company-specific information is not available in public training data.
  • Models may hallucinate facts when they lack information.
  • Retraining a model whenever information changes is impractical.

RAG addresses these challenges by giving the model access to current and domain-specific information at runtime.

The Core Idea

Many techniques used with LLMs improve performance by providing additional context.

Examples include:

  • Few-shot prompting
  • Tool usage
  • System instructions
  • Context injection

RAG extends this concept by automatically finding and supplying relevant information from a knowledge source.

The more relevant context provided to the model, the more accurate and useful its responses can become.

How RAG Works

A typical RAG workflow looks like this:

User Question
      ↓
Retrieve Relevant Information
      ↓
Add Information to Prompt
      ↓
Send Prompt to LLM
      ↓
Generate Response

The retrieval step happens before the model generates an answer.

Instead of asking the model to rely entirely on memory, we first provide it with the information needed to answer the question.

Knowledge Bases

A knowledge base is a repository of information that can be searched and retrieved.

Examples include:

  • Company documentation
  • Product catalogs
  • Internal wikis
  • Policy documents
  • Databases
  • SharePoint repositories
  • PDFs and reports

The knowledge base serves as the source of truth, while the LLM serves as the reasoning and language generation layer.

Example: Travel Assistant

Imagine a travel company storing information about:

  • Ticket prices
  • Destinations
  • Flight schedules
  • Local activities

A user asks:

What is the ticket price to London?

The application searches the knowledge base for information related to London and retrieves relevant records.

That information is added to the prompt before sending it to the LLM.

The model then generates an answer using the retrieved data, making it appear knowledgeable about the company's latest travel information.

A Simple RAG Implementation

Consider an insurance company with documents describing products and employees.

A very basic implementation might work as follows:

  • Scan all company documents.
  • Identify product names and employee names.
  • When a user asks a question, check whether any known names appear in the query.
  • If a match is found, retrieve the associated document.
  • Add the document contents to the prompt.
  • Allow the LLM to generate an answer.

While this approach is simplistic and not suitable for large-scale systems, it demonstrates the fundamental principle behind RAG: retrieve information first, then generate the response.

Key Takeaways

  • RAG stands for Retrieval Augmented Generation.
  • It combines information retrieval with LLM-based text generation.
  • Relevant information is retrieved before the model generates an answer.
  • The retrieved information is included as context in the prompt.
  • RAG helps reduce hallucinations and enables access to current and domain-specific knowledge.
  • Knowledge bases can contain documents, databases, policies, manuals, and other sources of information.
  • The fundamental idea behind RAG is simple: provide the model with the information it needs before asking it to answer.
