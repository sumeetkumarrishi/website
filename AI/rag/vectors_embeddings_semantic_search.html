Vectors, Embeddings, and Semantic Search

Category: AI → RAG

Introduction

One of the most important concepts in modern AI systems is the idea of embeddings and vectors. These concepts form the foundation of semantic search, vector databases, and Retrieval Augmented Generation (RAG).

At first glance, vectors may seem like a complex mathematical concept. In reality, they are simply a way of representing the meaning of text using numbers.

Understanding vectors is the key to understanding how modern AI systems search for information based on meaning rather than exact keywords.

Two Types of Language Models

Modern AI systems typically use two different kinds of models.

Autoregressive Models

These are the models most people interact with directly. Examples include:

  • GPT
  • Claude
  • Gemini
  • DeepSeek

Their primary job is to generate text. Given a sequence of words, they predict the next token. For example:

The capital of France is

The model predicts:

Paris

It then continues predicting one token at a time until a complete response is generated.

This process powers chatbots, assistants, code generation tools, and most conversational AI systems.

Embedding Models

Embedding models serve a completely different purpose. Instead of generating text, they analyze the input and produce a numerical representation of its meaning. For example:

How do I learn Python?

might become:

[0.12, -0.45, 0.88, 0.31, ...]

This collection of numbers is known as an embedding vector.

The goal of an embedding model is not to answer the question, but to capture the meaning of the text in a form that computers can compare efficiently.

What is a Vector?

A vector is simply a list of numbers. Examples:

[4, 2, 7]

[0.12, -0.45, 0.88]

Modern embedding models typically generate vectors containing hundreds or thousands of values. Examples include:

  • 384 dimensions
  • 768 dimensions
  • 1024 dimensions
  • 1536 dimensions
  • 3072 dimensions

Each number contributes to a mathematical representation of the text's meaning.

How Meaning Becomes Numbers

Consider the following sentences:

How do I learn Python?

Best way to study Python programming

Although the wording is different, the meaning is very similar.

When processed by an embedding model, both sentences produce vectors that are close together in mathematical space. For example:

How do I learn Python?
→ [0.12, -0.45, 0.88, ...]

Best way to study Python programming
→ [0.11, -0.48, 0.86, ...]

The exact values are not important. What matters is that similar meanings produce similar vectors.

Why Vectors Matter

Traditional search systems rely on matching keywords. For example:

Python learning

Study Python programming

contain different words. A keyword search may struggle to recognize that they are asking about the same topic.

Embedding models solve this problem by converting both phrases into vectors and comparing their meanings.

  • If two vectors are close together, the meanings are likely similar.
  • If two vectors are far apart, the meanings are likely different.

This allows AI systems to search by meaning instead of searching by exact words.

Semantic Search

Searching by meaning is known as semantic search.

Instead of asking:

Do these documents contain the same keywords?

semantic search asks:

Do these documents mean similar things?

This produces much better search results, especially when users use different wording than the original documents.

For example, a document might contain:

Python programming tutorial

A user might search for:

How can I learn Python?

Even though the words differ, semantic search can identify that both are discussing the same topic.

Vectors and RAG

Vectors are the foundation of modern RAG systems.

Imagine a company has 10,000 documents. Each document is processed through an embedding model and converted into a vector.

Document A → Vector A
Document B → Vector B
Document C → Vector C

Now a user asks:

How do I study Python?

The question is also converted into a vector.

Question → Vector Q

The system then compares Vector Q against all document vectors. The documents with vectors closest to the question are considered the most relevant.

These documents are retrieved and supplied to the LLM as context. This allows the model to answer using information that is relevant to the user's intent, even when the exact wording differs.

Tokens vs Vectors

A common source of confusion is the difference between tokens and vectors.

Tokens

Tokens are simply pieces of text represented as IDs. For example:

I love Python

might become:

[25, 981, 4521]

These numbers are identifiers used by the model. They do not directly represent meaning.

Vectors

After the text has been processed by an embedding model, it becomes:

[0.12, -0.45, 0.88, ...]

This vector contains semantic information. It represents the meaning of the text rather than the words themselves.

The process looks like this:

Text
 ↓
Tokens
 ↓
Embedding Model
 ↓
Vector

Tokens are the input representation. Vectors are the meaning representation.

The Foundation of Modern Retrieval

The key insight behind embeddings is simple:

Texts with similar meanings produce similar vectors.

Because of this property, computers can compare vectors mathematically and identify related information. This capability enables:

  • Semantic Search
  • Vector Databases
  • Retrieval Augmented Generation (RAG)
  • Document Retrieval Systems
  • Enterprise Knowledge Search
  • AI-Powered Search Engines

Nearly every modern retrieval system is built upon this idea.

Key Takeaways

  • Autoregressive models generate text by predicting the next token.
  • Embedding models convert text into numerical representations called vectors.
  • A vector is simply a list of numbers.
  • Similar meanings produce similar vectors.
  • Semantic search compares vectors rather than keywords.
  • Vector search allows systems to retrieve relevant information even when wording differs.
  • Modern RAG systems use embeddings and vector search to find relevant documents before generating answers.
  • The foundation of semantic retrieval is the idea that similar meanings occupy nearby positions in vector space.
