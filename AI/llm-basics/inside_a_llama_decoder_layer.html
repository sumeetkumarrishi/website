Inside a Llama Decoder Layer

Category: AI → LLM Basics

Introduction

A large language model is built by stacking many identical decoder layers. Understanding what happens inside a single layer is the key to understanding the whole model.

This walkthrough traces one complete decoder layer step by step. The concrete numbers used below describe a small example model (a compact Llama with 16 layers, a hidden size of 2048, a feed-forward size of 8192, and a vocabulary of 128,256 tokens).

1. Self-Attention

The model creates three projections of the input:

Q = X × Wq
K = X × Wk
V = X × Wv

Meaning:

Q = What information do I need?
K = What information can I provide?
V = The actual information I contain.

The model calculates attention scores and determines:

How much information should flow
between tokens?

Example:

Paris pays attention to:
France  → 80%
capital → 15%
others  → 5%

The result is a new context-aware vector.

2. o_proj (Output Projection)

After attention finishes, we have an attention output. In this example model:

o_proj:
2048 → 2048

Purpose:

Combine everything learned by attention
into a single clean representation.

Think of it as:

Attention gathered information. o_proj organizes that information.

Human analogy:

Several researchers gathered findings.
o_proj writes the final report.

3. Residual Connection

The model does not throw away the original input. Instead:

Output =
Original Input
+
Attention Output

Example:

Original Paris Vector
+
New Information About France

Purpose:

Keep original knowledge
while adding new knowledge.

Without residual connections:

Information could be lost
as layers become deeper.

4. RMSNorm

Before entering the next stage, the model applies RMSNorm.

Purpose:

Keep vector values stable.

Human analogy:

If some numbers become too large
or too small,

RMSNorm brings them back
into a healthy range.

Think:

Attention works better when values stay balanced.

5. MLP (Multi-Layer Perceptron)

Now the token already knows the context. The MLP asks:

What should I do with this information?

In this example model:

Input: 2048
      ↓
gate_proj: 2048 → 8192

up_proj:   2048 → 8192

SiLU Activation

down_proj: 8192 → 2048

What MLP Does

Attention:

Collects information.

MLP:

Processes information.

Example — after attention:

Paris knows:

- France exists
- Capital relationship exists

MLP may strengthen concepts like:

Paris = city
Paris = capital city
Paris = capital of France

Think:

Attention = Gathering facts. MLP = Thinking about those facts.

6. Second Residual Connection

Again:

Output =
Previous Output
+
MLP Output

Purpose:

Preserve earlier knowledge
while adding refined understanding.

Attention:
"Let me gather relevant information."

o_proj:
"Let me organize that information."

Residual:
"Don't forget what I already knew."

RMSNorm:
"Keep everything stable."

"Let me think about the information."

"Repeat the process with a better representation."
