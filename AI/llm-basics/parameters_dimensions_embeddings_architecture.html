Understanding Parameters, Dimensions, Embeddings, and Architecture in Large Language Models

Category: AI → LLM Basics

Introduction

When people first start learning about Large Language Models (LLMs), they often encounter terms such as parameters, dimensions, embeddings, and transformers. These terms are related, but they represent different concepts.

This article explains these concepts from first principles and shows how they work together inside an LLM.

What is a Parameter?

A parameter is a learned numerical value inside a neural network.

Think of a parameter as a small adjustable knob that influences the behavior of the model.

Example:

0.734    -1.221    2.448

When people say a model has 8 billion parameters, they mean the model contains approximately 8,000,000,000 learned numbers.

A parameter is not a fact or a property. Instead, parameters are numerical settings that collectively enable the model to learn and represent knowledge.

Are Parameters Created During Learning?

No.

Parameters exist before training begins. The architecture creates parameter slots first, and training learns the values stored in those slots.

Architecture → Creates parameter slots → Training → Learns parameter values

What Does a Parameter Correspond To?

A parameter usually corresponds to the strength of a mathematical relationship or transformation inside the network. Examples:

  • How strongly one dimension influences another
  • How strongly one pattern activates another
  • How much one token should pay attention to another

A parameter does not directly correspond to concepts such as Dog, Paris, Barking, or JavaScript.

Why a Single Parameter Has No Meaning

A single parameter value like 0.734 means almost nothing by itself.

Meaning emerges from patterns across billions of parameters, much like a color emerges from the combination of RGB values.

Knowledge is not stored in individual parameters. It emerges from patterns across many parameters.

What is a Dimension?

A dimension is one coordinate in a vector. Example:

[0.8, -0.3, 1.2, 0.7]

This vector has 4 dimensions.

Modern LLMs commonly use vectors containing thousands of dimensions such as 4096, 8192, or 12288.

Parameters vs Dimensions

Dimensions define the size of the representation space. Parameters are the learned numerical values used to transform vectors.

Dimensions = Coordinates in the space
Parameters = Learned transformations between coordinates

What is an Embedding?

When the word "dog" enters a model, it is converted into a vector:

Dog = [0.8, -0.3, 1.2, 0.7]

This vector is called an embedding.

An embedding is the model's numerical representation of a concept.

The RGB Analogy

RGB values:

[255, 0, 0]

represent the color red.

No single value means "red". The combination of values creates the color.

Similarly, no single embedding dimension means "fur", "tail", or "barks". Meaning emerges from the pattern of values.

Do Embedding Dimensions Have Names Like RGB?

For RGB:

Dimension 1 = Red
Dimension 2 = Green
Dimension 3 = Blue

For embeddings:

Dimension 1 = ?
Dimension 2 = ?
Dimension 3 = ?

Most of the time, nobody knows. The model learns these dimensions automatically during training.

Latent Features

Researchers often call embedding dimensions latent features or latent variables. Latent means hidden and not directly observable.

Some dimensions may partially correlate with concepts such as sentiment, geography, grammar, or programming, but their meanings are usually not cleanly separated.

Distributed Representation

Modern neural networks use distributed representations. Instead of:

Dimension 1 = Animal
Dimension 2 = Fur
Dimension 3 = Bark

A concept is represented by patterns spread across many dimensions.

Meaning is distributed rather than stored in one location.

Where is Knowledge Stored?

Knowledge is distributed across many dimensions and many parameters. The model does not store:

Dog: Barks = True

Instead, relationships between concepts such as dog, bark, pet, and animal are encoded as patterns in vectors and parameters.

What Happens When You Type a Word?

dog
 → Token ID
 → Embedding Vector
 → Transformer Layers
 → Probability Calculation
 → Next Token Prediction

The model's primary job is to predict the next token.

Why Aren't Embeddings Enough?

Embeddings represent concepts. Transformer layers learn how concepts interact. Examples:

  • Dog + Bark
  • Dog + Pet
  • Paris + France
  • JavaScript + Loop

Without transformer layers, the model could represent concepts but would struggle to reason about them.

Vectors and Matrices

Vector:

[1, 2, 3]

Matrix:

[ [1, 2, 3],
  [4, 5, 6] ]

A matrix can be thought of as an array of vectors.

How LLMs Use Matrices

A single word becomes a vector. A sentence becomes a matrix made up of multiple vectors.

Transformers process these matrices using large matrix multiplications, which is why GPUs are highly effective for AI workloads.

Relationship Between Dimensions and Parameters

Dimensions and parameters are related but not the same thing. Example:

4096 dimensions

may require a matrix:

4096 x 4096

which contains:

16,777,216 parameters

More dimensions generally lead to more parameters. However:

Parameter Count != Dimension Count

Can You Determine Dimensions from Parameter Count?

Usually no. Parameter count depends on:

  • Hidden dimensions
  • Number of layers
  • Feed-forward size
  • Vocabulary size
  • Attention structure

Different architectures can have the same parameter count while using different dimensions.

What is Model Architecture?

Architecture is the blueprint of the model. It defines:

  • Number of layers
  • Hidden dimensions
  • Attention structure
  • Feed-forward structure
  • Vocabulary size

Architecture creates parameter slots. Training learns parameter values.

Architecture = Building Blueprint
Parameters   = Learned Settings Inside The Building

Key Takeaways

  • Parameters are learned numerical values.
  • Parameters represent learned strengths of mathematical transformations.
  • Individual parameters have little meaning by themselves.
  • Dimensions define the size of vector representations.
  • Embeddings convert words into vectors.
  • Embedding dimensions do not usually have human-readable names like RGB.
  • Dimensions are often called latent features.
  • Knowledge is distributed across many parameters and dimensions.
  • Vectors represent concepts.
  • Matrices transform concepts.
  • Transformer layers use billions of parameters to reason about relationships.
  • Architecture defines the model structure before training begins.
  • Training adjusts parameter values to create the final model.

The most important insight is that an LLM does not store knowledge as database fields or objects. Knowledge emerges from patterns distributed across vectors and billions of learned parameters.
